# 05. 既存CSV実形式確認（Phase 1-1）

## 確認対象フォルダ

```
~/Documents/さいはて/G検定クロスストーリー/
```

## 対象CSV

`G検定_4択問題ファイル1_100問.csv` 〜 `G検定_4択問題ファイル10_100問.csv`（10ファイル）

## 総件数

- 1000問（全10ファイル合計）
- 各ファイルとも100問ずつ

## 文字コード

- 全ファイル UTF-8（BOM付き）

### 読み込み時の注意

- BOMを除去すること
- JavaScript/TypeScript側でヘッダーの先頭が `﻿id` にならないよう注意すること（BOMがidキーに混入すると、`row.id` が `undefined` になる等の不具合につながる）

## ヘッダー構成

**基本形**（10ファイル中9ファイルが該当）：

```
id,分野,難易度,問題,A,B,C,D,正解,解説,出題済み
```

**例外**：

- `G検定_4択問題ファイル5_100問.csv` のみ `出題済み` 列がなく、10列構成

## 必須列

`id, 分野, 難易度, 問題, A, B, C, D, 正解, 解説`

## 必須列空欄

なし（全1000行・全必須列で空欄を確認せず）

## 正解列

値はすべて `A` / `B` / `C` / `D` のいずれか（それ以外の値・空欄なし）

## id重複

なし（全1000行でユニーク）

## セル内改行

なし

## セル内カンマ

- 5件あり（ReLUの `max(0,x)` を含む解説文で確認）
- 該当セルはCSVとして正しくダブルクォートで囲まれており、パース自体は問題なく成功する
- **そのため文字列の `split(",")` によるパースは禁止。CSVパーサー（例：PapaParse）を使うこと**

## 難易度の値

`標準` / `やや難` / `難` / `難しめ` の4種類（表記ゆれあり：「難」と「難しめ」が別表記で混在）

## 分野の値

50種類以上の値が存在し、表記ゆれ・粒度のばらつきが大きい（例：「AI史」「機械学習」のような大分類と、「Attention」「RAG」「Embedding」のような小分類が同じ列に混在）。
Phase 1ではこれを正規化せずそのまま扱い、将来的に正規化やsearch_tagsでの補完を検討する。

## Phase 1内部データ型への変換方針

| 既存CSV列 | 内部データ型フィールド |
|---|---|
| id | id |
| 分野 | category |
| 難易度 | difficulty |
| 問題 | question |
| A / B / C / D | choices |
| 正解 | answer |
| 解説 | explanation |
| （ファイル名） | sourceFile |
| 出題済み | usedStatus（列がない場合は空文字） |

## Phase 1での結論

- 既存CSVはそのまま読み取り対象にできる
- ただしBOM除去とCSVパーサーの利用は必須
- 標準CSV v1への完全変換はまだ行わない
