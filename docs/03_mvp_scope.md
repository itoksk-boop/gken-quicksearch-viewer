# 03. MVPスコープ

Phase 1（ブラウザ版プロトタイプ）で作るものと作らないものを明確にする。ここに書かれていない機能は、明示的な合意がない限りMVPに含めない。

## 既存CSVの扱い

現在確認されている既存CSV形式：

```
id,分野,難易度,問題,A,B,C,D,正解,解説,出題済み
```

標準CSV v1への完全な変換は行わず、Phase 1では以下の簡易マッピングで検索対象データを作る。

| 既存列 | マッピング先 |
|---|---|
| id | id |
| 分野 | category |
| 難易度 | notes または difficulty |
| 問題 | question |
| A | choice_a |
| B | choice_b |
| C | choice_c |
| D | choice_d |
| 正解 | answer |
| 解説 | explanation |
| 出題済み | notes |

このマッピングはPhase 1限定の暫定対応であり、標準CSV v1への正式な変換は別タスクとして扱う。

## MVPで作るもの

- 既存CSVまたはJSONの読み込み（起動時に一括読み込み）
- 既存CSV形式の簡易マッピング（上記表のとおり）
- 検索窓
- 2文字以上でのリアルタイム検索
- 半角/全角スペース区切りによる複数キーワード検索
- メモリ上N-gram検索（起動時にインデックスを構築し、検索時はDBを介さない）
- 予測候補の基本表示（グレー予測の最小実装）
- 上位4件のカード表示
- 5件目以降のリスト表示
- iPad横向きを想定したレイアウト

## MVPで作らないもの

- SQLiteによる保存
- Capacitor化
- Xcodeでの実機ビルド
- 標準CSV v1への完全変換
- ChatGPTによるterm/search_tagsの補完
- 重複判定の自動化（ラベル付けの自動生成）
- 学習履歴の記録
- 正答率の管理
- Mac版・Windows版

## MVPの成功基準

「01_concept.md」に定めた通り、人間が入力してから判断するまでの時間が最短になっているかどうかで評価する。機能の網羅性ではなく、2〜4件のカードが的確に、かつ即座に表示されることを優先する。
