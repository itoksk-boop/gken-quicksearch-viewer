# 04. Phase 1 実装順計画

## Phase 1の目的

iPad向け検索体験の検証である。React + TypeScript + Viteによるブラウザ版プロトタイプのみを対象とし、Capacitor / Xcode / SQLiteはまだ対象外とする（詳細は `02_technical_decision.md` を参照）。

本ドキュメントは、Phase 1の実装に着手する際の順序を固定するものであり、この順序自体は本指示の時点ではまだ実行しない。

## 実装順

### 1. 既存CSVの実形式確認

- 10個のCSVの列名、文字コード、行数、サンプル行を確認する
- この段階ではCSVを変更しない

### 2. 内部データ型の最小定義

- `id`, `category`, `difficulty`, `question`, `choices`, `answer`, `explanation`, `sourceFile` を最低限とする
- 標準CSV v1の全項目はまだ使わない

### 3. CSV→内部データ変換

- 既存CSV形式を最小内部データ型へ変換する
- `answer` はA/B/C/Dに正規化する
- 変換できない行はログに出す

### 4. メモリ上N-gram検索

- 検索時にDBは使わない
- 2文字以上で検索する
- 半角/全角スペースで複数キーワードに対応する
- 上位4件を返す前提でスコアリングする

### 5. グレー予測候補

- まずはterm/search_tagsなしでも出せる簡易候補から始める
- 問題文・分野・解説から候補語を抽出する案を検討する

### 6. 最小UI

- 検索窓
- グレー予測の基本表示
- 上位4件カード
- 5件目以降リスト
- iPad横向き想定

### 7. iPad Safari確認

- 文字サイズ
- 4カードの見え方
- ソフトウェアキーボード
- 日本語入力
- Enter確定とIME変換の衝突

## MVPでまだ作らないもの

- SQLite
- Capacitor
- Xcode実機ビルド
- 標準CSV v1完全変換
- 重複判定自動化
- 学習履歴
- 正答率管理
- Mac/Windows版
