# 02. 技術方針

本ドキュメントは着手前調査（技術選定・実現性調査）の結論を踏まえた、現時点での技術方針を確定するものである。実装フェーズを明確に分け、各フェーズで「やること／やらないこと」を固定する。

## フェーズ構成

### Phase 1：ブラウザ版プロトタイプ

- **React + TypeScript + Vite** によるブラウザ版プロトタイプとする
- Capacitor / Xcode / SQLite にはまだ入らない
- Phase 1の目的は検索体験の検証である。具体的には以下が成立するかを確かめる
  - 2文字以上のリアルタイム検索が実用速度で動くか
  - メモリ上N-gram索引による検索結果が「求めていた4件」を正しく上位に出せるか
  - グレー予測・Enter確定・タップ確定のUXが違和感なく動くか
- ネイティブアプリ化・実機確認・データ永続化の課題は本フェーズでは扱わない

### Phase 2以降：ネイティブ化・永続化の検討

- Capacitor + SQLite の採用をここで検討する（Phase 1で検索体験が固まったのちに着手）
- **SQLiteは保存用**であり、検索時にDBを叩く設計にはしない
- 検索本体は、起動時に生成する**メモリ上N-gram Key-Value Index**とする（Phase 1〜2を通じて一貫させる方針）
- SQLiteはCSV取り込み・重複管理・バックアップ/エクスポートの基盤として位置づけ、起動時に全件をメモリへ展開したあとは検索処理から一切参照しない

## PWAの位置づけ

PWAは最終版の配布形態としない。Xcodeなしで実機（iPad Safari）から素早く動作確認できる**試作用の手段**として位置づける。iOS Safariのストレージ消去仕様やIndexedDBの安定性の懸念から、本番の保存基盤としては採用しない。

## 採用しない技術（現時点）

- **Excel VBA**：採用しない
- **Tauri / Electron**：iPad版が対象外である現時点では検討自体を行わない
- **Mac版 / Windows版の技術基盤**：現時点では一切決めない。Phase 1・Phase 2の意思決定に影響させない

## 検索方式の確定事項

- 検索本体はメモリ上N-gram（バイグラム/トライグラム）Key-Value Indexとする
- SQLite FTS5・単純includes・Fuse.js等は主エンジンとして採用しない（着手前調査を参照）
- グレー予測候補の生成は、全文検索用のN-gram索引とは別に、用語・タグの前方一致構造を用意して対応する
